# Model-driven App

## Purpose

The Model-driven app provides operational visibility over the Access Requests lifecycle.

## Suggested Views

- Active Access Requests
- Pending Approval
- Approved
- Rejected
- Fulfilled
- Overdue / Reminder Sent
- Escalated
- Records With Errors

## Suggested Form Sections

- Request details
- Requester identity
- Approval details
- CaseSummary and CorrelationId
- Fulfilment / resolution
- SLA reminders and escalation
- Error capture
- Audit history

## Governance Value

The Model-driven app helps approvers, fulfilment/operations users, and admins review lifecycle state without relying only on Power Automate run history. It supports visibility into Status, ApproverDecision, FulfillmentNotes, ApprovalError, FulfillmentError, ReminderCount, EscalationCount, and Dataverse audit history.
